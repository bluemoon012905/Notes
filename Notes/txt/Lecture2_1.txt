36

Benefits of MVC Architecture
The loosely coupled feature (compared with ASP .Net architecture)
among the tiers in Views, Models, and Controllers:
’ü« Allow team members to develop different parts of the application
with reduced interactions: GUI and code are separated;
’ü« The template can generate unit tests by automatically adding
mock objects (simulated objects) that mimic the behavior of other
objects in controlled ways in the application;
’ü« The restricted use of the view state, page postback, and related
server controls, such as Gridview, allows the framework to render
the XHTML page less dependent on the server features;
’ü« The MVC applications are more friendly with URI-based access
used for RESTful services, the feed data formats, and cloud
computing environment (Unit 5).

How are View and Controller Connected?

46

In ASP .Net Framework, the .aspx files and .aspx.cs
files are placed together in the same folder. The links
are implicit. You must double-click the buttons to
create the implicit links. It works because the two
pages are tied together.
’ü« In MVC, the links are explicit.
’ü« Each View component (.aspx file) in browser
contains a list of JavaScript calls.
’ü« The Controller of the component contains the
methods to be called. View page and .cs page can
be completely separated.
