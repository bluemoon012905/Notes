Dataverse Development Template

8

https://learn.microsoft.com/en-us/power-apps/maker/data-platform/data-platform-intro




This template focus on data-oriented app development.
Microsoft Dataverse: is a virtual cloud datastore
consisting of tables of rows and columns.






It is seamlessly integrated in all components of Power
Platform, as well other platforms, such as Microsoft Azure
and Microsoft 365 infrastructure and Dynamic 365 apps.
It allows the integration of different data sources at datastore
layer, instead of dealing them at application logic layer.
Note: Excel and SQL are concrete stores. Dataverse presents
an abstract layer above the concrete stores.



Model-Driven Apps Development Template

10

https://learn.microsoft.com/en-us/power-apps/maker/model-driven-apps/model-driven-app-components



Model-driven apps typically consists of the following
components: Data, UI, logic, and visualization:
Data: Tables, relationship among tables, columns of a table.
Dataverse is applied in this model.
 UI: It determines how human users interact with the app,
e.g., Site map, Form, View, Custom page, …
 Logic: Business process flow, workflow, actions, business
rules. Power Automate can be applied for model-driven
logic.
 Visualization: It determines what type of data visualizations
and reporting that the app will have available, e.g., Chart,
Dashboard. Power BI can be applied here.


What is Power Automate?

28

Power Apps create individual apps using composition
constructs and data sources.
 Power Automate is a workflow service that automates
actions across the most common apps and services.
 For example, you can create a flow that posts a message
on a Microsoft Teams channel whenever an selected
event occurs in your favorite planning tool. These types
of integrations can help the various tools you use
everyday work better together.
 You can connect to hundreds of services and can
manage data in the cloud or in on-premises sources.


Y. Chen

Creating Your Workflow in Desktop Flow
1. Create a new desktop flow in Power Automate Desktop
(https://learn.microsoft.com/en-us/power-automate/desktop-flows/introduction)
2. Add actions and set up configuration.
3. Define inputs and outputs.
4. Test the desktop flow you created.
5. If needed, set up triggers and links between cloud
flows and Power Automate Desktop.
Y. Chen

31

Creating Your Workflow in Cloud
1. Create a cloud flow in Power Automate
(https://make.powerautomate.com/)
2. Specify an event or trigger to start the flow
with connectors. Based on the way you want to
trigger the automation: You'll specify what event you
want to use to trigger your Power Automate flow.
3. Add actions and conditions.





Microsoft Power Pages allows you to create and design a
business website using templates quickly without a
professional developer.
It is a secure, enterprise-grade, low-code software as a service

modern external-facing business websites.
Whether you are a low-code maker or a professional

Copilot Studio

52

Copilot Studio is a graphical and low-code tool.
 It can be used for creating agents (AI services).
 An agent is a powerful AI companion that can handle
a range of interactions and tasks, e.g., resolving issues
requiring complex conversations, autonomously
determining the best action to take based on its
instructions and context.
 Agents can engage with customers and employees and
coordinate a collection of language models, along
with instructions, context, knowledge sources, topics,
actions, inputs, and triggers to accomplish your
desired goals.

